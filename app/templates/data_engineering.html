Feature Engineering — Informality Index

📐 Formula

informality_index = 1 − mean(RUT_bin, CC_bin, ARL_bin, has_accounting)

Range: 0 (fully formal) → 1.0 (fully informal). Each flag = 1 if the business HAS the formal attribute, 0 if not.

🔑 Flag Variables

  • ◆P1633_bin — Has RUT (tax ID): 1=Yes, 0=No
  • ◆P1055_bin — Chamber of Commerce: 1=Yes, 0=No
  • ◆P3091_bin — ARL contribution: 1=Yes, 0=No
  • ◆has_accounting — Keeps records (P640 ≠ 5): 1=Yes, 0=No

Normalization & Encoding

⚖️
StandardScaler Applied to P3034 (workers), P3035 (months in operation), P3088 (monthly sales). Produces mean≈0, std≈1 for each numeric variable.
🔢
One-Hot Encoding pd.get_dummies applied to P3053 (CIIU sector). Drop_first=True to avoid multicollinearity. Each category becomes a binary column.
🧹
Null Imputation Categorical variables: mode imputation. Numerical variables: median imputation by sector and city — prevents bias from non-response patterns.
📝
Documented Pipeline All steps logged with before/after shape. Reproducible via the data_engineering_controller.py — re-run by calling GET /data-engineering.
