Data Cleaning & Null Value Handling

🧹 Cleaning Steps

  • ◆Dropped survey identifier columns (DIRECTORIO, SECUENCIA_P, SECUENCIA_ENCUESTA, ORDEN) — not features, only used for data collection tracking.
  • ◆Removed records with EDAD < 18 (under-age business owners excluded per EMICRON methodology).
  • ◆Standardized department codes to match official DANE DIVIPOLA nomenclature.
  • ◆Removed CIIU codes excluded by EMICRON scope (divisions 35, 36, 51, 84, 97–99).

🔧 Null Value Strategy

  • ◆Categorical variables (P1633, P1055, P3091, P640, DPTO): mode imputation — replaces with the most frequent value within the same department group.
  • ◆Numerical variables (P3034, P3035, P3088): median imputation by sector and city — robust to outliers and preserves the distribution shape.
  • ◆P3088 (monthly sales) had the highest missing rate (~7%). Median was computed separately per CIIU sector to avoid cross-sector contamination.

Feature Engineering — Informality Index

📐 Formula

informality_index = 1 − mean(RUT_bin, CC_bin, ARL_bin, has_accounting)

Range: 0.0 (fully formal) → 1.0 (fully informal). Each flag = 1 if the business HAS the formal attribute, 0 if not. The composite index summarizes formality compliance in a single interpretable score.

🔑 Component Flags

  • ◆P1633_bin — Has RUT: 1=formal, 0=informal
  • ◆P1055_bin — Chamber of Commerce: 1=formal, 0=informal
  • ◆P3091_bin — ARL contribution: 1=formal, 0=informal
  • ◆has_accounting — Keeps records (P640 ≠ 5): 1=formal, 0=informal

Data Transformation — Encoding & Normalization

🔢
Binary Encoding (formality flags) P1633, P1055, P3091 recoded from {1,2} to {1,0} — 1=has the formal attribute, 0=does not. Makes these variables directly usable as numeric features in K-Means.
📊
Ordinal → Binary (accounting) P640 (5 categories) collapsed to binary: values 1–4 → 1 (keeps some records), value 5 → 0 (no records). Simplifies without losing informality signal.
⚖️
StandardScaler (normalization) Applied to P3034 (workers), P3035 (months), P3088 (sales). Produces mean≈0, std≈1. Essential for K-Means which is distance-based and sensitive to scale differences.
🎯
One-Hot Encoding (CIIU sector) pd.get_dummies on P3053 with drop_first=True to avoid multicollinearity. Each economic sector becomes a binary dummy column for the clustering algorithm.
