📐 Dataset Split & Feature Engineering

Split strategy: 80/20 train-test split with stratification on the target variable to preserve class balance. Random state fixed at 42 for reproducibility.


Target variable: INFORMALITY_INDEX ≥ 0.75 → 1 (high informality), <0.75 → 0 (lower informality). Binary classification derived from the composite index.


Preprocessing: MONTHLY_SALES log-transformed (log1p) to reduce skewness. All features normalized with StandardScaler (mean=0, std=1) before training supervised models.

Model 01 · Unsupervised

K-Means Clustering

K-Means is an unsupervised algorithm that groups data points into k clusters by minimizing within-cluster variance (inertia). It requires no labeled data — ideal for discovering hidden informality patterns in EMICRON without predefined categories. Applied to all 81,021 records.

⚙️ Hyperparameters

n_clusters = 4 — selected via Elbow Method + Silhouette Score analysis
n_init = 10 — runs 10 initializations, keeps best result
random_state = 42 — reproducible centroids
algorithm = lloyd — standard Lloyd's algorithm

Model 02 · Supervised Classification

Logistic Regression

Logistic Regression models the probability of a binary outcome using a linear combination of features passed through a sigmoid function. It is interpretable, fast, and serves as a baseline supervised model to classify micro-businesses as high-informality (1) or lower-informality (0).

⚙️ Hyperparameters

C = 1.0 — regularization strength (inverse). Higher C = less regularization
solver = lbfgs — Limited-memory Broyden–Fletcher algorithm
max_iter = 500 — maximum convergence iterations
class_weight = balanced — adjusts for 80/20 class imbalance

Model 03 · Ensemble Learning

Random Forest Classifier

Random Forest builds multiple decision trees on random subsets of data and features, then aggregates their predictions (majority vote). It handles non-linear relationships, is robust to outliers, and provides feature importance scores — ideal for understanding which variables drive informality classification.

⚙️ Hyperparameters

n_estimators = 100 — number of decision trees in the forest
max_depth = 8 — limits tree depth to prevent overfitting
class_weight = balanced — handles 80/20 class imbalance
random_state = 42 — reproducible results
