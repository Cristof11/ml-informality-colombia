Methodology

CRISP-ML(Q)

Cross-Industry Standard Process for Machine Learning with Quality Assurance — the structured framework guiding this project from problem definition to deployment.

"CRISP-ML(Q) extends the well-known CRISP-DM process model to address the specific requirements of machine learning projects, adding a quality assurance layer to each phase."

— Symeon Symeonidis et al. (2022). A CRISP-ML(Q) Reference Checklist for Responsible ML.

The 6 Phases

🎯
Phase 01 · Active

Business Understanding

Define the ML problem in business terms. Identify stakeholders, establish success criteria, assess feasibility and document risks. For this project: characterize commercial informality patterns in Colombian cities using open EMICRON data.

View Phase →
📊
Phase 02 · Active

Data Understanding

Explore the EMICRON 2023 dataset: variable analysis, missing value assessment, statistical summaries, distribution analysis, and correlation matrix across 81,018 micro-business records in 24+ departments.

View Phase →
⚙️
Phase 03 · Active

Data Engineering

8-step pipeline: drop ID columns → feature selection → null imputation → binary encoding of formality flags → accounting records encoding → Informality Index engineering → StandardScaler normalization → One-Hot Encoding of CIIU sector.

View Phase →
🤖
Phase 04 · Pending

Model Engineering

Select, configure and train the K-Means clustering model. Determine optimal k via Elbow Method and Silhouette Coefficient. Apply PCA for dimensionality reduction and visualization.

✅
Phase 05 · Pending

Model Quality

Evaluate clusters with Silhouette Score (target ≥ 0.4), Davies-Bouldin Index, and WCSS. Interpret cluster profiles in business terms. Validate against known GEIH city-level informality rates.

🚀
Phase 06 · Pending

Deployment & Monitoring

Deploy the trained pipeline as a REST API endpoint. Monitor cluster drift as new annual EMICRON data is released. Update model yearly.
